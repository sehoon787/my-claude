Core Principles 핵심 원칙

Three ideas that make the orchestration reliable at scale.

👑

Leadership

Boss orchestrates, never implements. Leads teams with peer-to-peer communication, dynamic composition, and file ownership protocols.

🔍

Discovery

Runtime capability matching — no hardcoded routing. Every agent, skill, and MCP server is auto-discovered at session start.

✅

Verification

Trust but verify. Every subagent result is independently checked before acceptance. No self-approval in the same context.

How Boss Works 보스의 작동 방식

Boss is the dynamic meta-orchestrator. It discovers all installed components at runtime and routes tasks through a 4-priority chain.

H

Harness

Runtime platform (OS). Claude Code, omo

O

Orchestrator

Coordinates agents. Never writes code. Boss, Sisyphus, Atlas

A

Agent

Performs actual work. Debugger, executor, reviewer

4-Priority Routing Chain

P1 — Exact Skill Match

"merge PDFs" → Skill("pdf")

no match ↓

P2 — Specialist Agent Match

"security audit" → Agent("Security Engineer")

no match ↓

P3 — Direct / Sub-orchestrator

P3a: Boss parallel (2-4 tasks) · P3b: Sisyphus (complex) · P3c: Agent Teams

no match ↓

P4 — General Fallback

"explain this" → Agent(model="sonnet")

Skill vs Agent: 3D Conflict Resolution

When both a Skill and Agent match, Boss scores across three dimensions: Scope, Depth × 2 (double-weighted), and Interactivity. The higher score wins — no hardcoded priority tables.

Quick Install 빠른 설치

One command to install everything.

git clone --depth 1 https://github.com/sehoon787/my-claude.git /tmp/my-claude && bash /tmp/my-claude/install.sh && rm -rf /tmp/my-claude

Or install as a Claude Code plugin:

/plugin marketplace add sehoon787/my-claude
/plugin install my-claude@my-claude
